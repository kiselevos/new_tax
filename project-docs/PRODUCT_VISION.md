# new_tax - Product Vision

**Версия:** 1.0
**Дата:** апрель 2026

## Проблема

Большинство россиян не понимают, сколько на самом деле стоит их труд. Расчётный листок показывает «к выплате» - число, которое уже прошло через цепочку удержаний. Переход на прогрессивную шкалу в 2025 году добавил сложности: пять ставок, северные надбавки, отдельная база для северной надбавки - это система, которую невозможно посчитать в уме.

Бухгалтеры и HR используют платные корпоративные системы. Рядовой сотрудник или соискатель не имеет инструмента, который даёт точный ответ бесплатно и прозрачно.

## Для кого

**Сегмент 1 - Работающий россиянин (основной)**
Хочет понять: сколько налогов удерживается с его зарплаты, попадает ли он под повышенную ставку, сколько реально стоит работодателю. Использует раз в год или при смене работы.

**Сегмент 2 - HR и рекрутер**
Считает стоимость сотрудника для бюджета. Нужна помесячная детализация с взносами работодателя. Использует при формировании офферов и бюджетировании.

**Сегмент 3 - Разработчик / интегратор**
Встраивает расчёт НДФЛ в свой продукт через API. Нужна стабильная точная машина расчёта с документированным интерфейсом.

## Что проект делает

- Рассчитывает НДФЛ по прогрессивной шкале 2025 года с корректным алгоритмом YTD (накопительный итог)
- Поддерживает районные коэффициенты и северные надбавки как отдельные налоговые базы
- Показывает взносы работодателя (ПФР, ФОМС, ФСС) с учётом годовых лимитов
- Предоставляет UI и API для программного доступа
- Открытый код - алгоритм можно проверить

## Что проект не делает (и не будет делать)

- Не является юридически значимым документом
- Не учитывает налоговые вычеты (имущественный, социальный, стандартный на детей) - это следующий этап
- Не поддерживает ИП, самозанятых, дивиденды
- Не хранит персональные данные пользователей

## Метрики успеха

- Корректность: расчёты совпадают с официальными бухгалтерскими системами на тестовых кейсах
- Актуальность: ставки и лимиты обновлены до начала нового налогового года
- Доступность: сервис работает без Redis, без внешних зависимостей (graceful degradation)

## Горизонт планирования

**2025 (текущий год):** стабилизация расчётного ядра, покрытие тестами всех веток, документация API.

**2026:** обновление ставок и лимитов под новый налоговый год, добавление налоговых вычетов (стандартный вычет на детей - наиболее востребован), публичный API с версионированием.
