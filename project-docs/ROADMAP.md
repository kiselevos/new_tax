## Фаза 5 — Подготовка к релизу (текущий приоритет)

**Цель:** выпустить стабильный калькулятор для ТД, ГПХ и самозанятых.

### 5.1 Блок вычетов для самозанятых
- [ ] На странице результата: аккордеон «Налоговые вычеты» для НПД
- [ ] Внутри: объяснение что стандартные вычеты НДФЛ недоступны при НПД
- [ ] Показывать использованный регистрационный бонус 10 000 ₽ (сколько осталось)

### 5.2 Единый файл налоговых констант в коде
- [ ] Создать `internal/calculate/constants_meta.go` — все числа с комментариями-источниками
- [ ] Убрать хардкод цифр из инфо-страниц, подтягивать через template functions
- [ ] Агент `tax-updater` обновляет `TAX_CONSTANTS.md` → изменения вносятся в один файл кода
- [ ] Проверить актуальность всех цифр на инфо-страницах (пример: «53 658 ₽ в 2025»)

### 5.3 Обновление API-страницы ✅
- [x] Добавить эндпоинт с `employment_type` (GPH, SELF_EMPLOYED)
- [x] Добавить поля НПД: `npd_income_source`, `has_registration_deduction`
- [x] Обновить примеры запросов/ответов

### 5.3.1 Swagger UI (отложено)
Варианты на выбор:
- **Полноценный Swagger UI** — написать OpenAPI 3.0 spec (`openapi.yaml`), подключить Swagger UI с CDN. Даёт интерактив и стандарт, но требует поддержки spec отдельно от proto.
- **Таблицы полей** — доработать текущую страницу: типы, описания, enum-значения. Без зависимостей, проще в поддержке.

### 5.4 Чистка главной страницы ✅
- [x] Оставить только «открытые формулы» и «не сохраняется на сервере»

### 5.5 Типы занятости в футере ✅
- [x] Добавить ссылку «Типы занятости» в навигацию футера

### 5.6 Обновление документации
- [ ] `ABOUT.md` — отразить ГПХ и НПД
- [ ] `CORE_MECHANICS.md` — описать логику НПД, регистрационный вычет

## Не берём (пока)

- Юридические лица — другой продукт
- Продажа имущества, аренда недвижимости — низкий приоритет
