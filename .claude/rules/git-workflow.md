# Git Workflow

## Ветки

- `main` — стабильная версия, только через PR
- `feature/<название>` — новые фичи
- `fix/<название>` — исправления багов

Текущая рабочая ветка: `feature/V2`

## Коммиты

Формат сообщения:
```
<тип>: <краткое описание на русском>
```

Типы:
- `feat` — новая функциональность
- `fix` — исправление бага
- `docs` — документация
- `chore` — технические изменения (агенты, конфиги, зависимости)
- `test` — тесты
- `refactor` — рефакторинг без изменения поведения

Примеры:
```
feat: добавлен расчёт налогового вычета на детей
fix: исправлено округление при переходе через порог ПФР
docs: обновлён ROADMAP, добавлена фаза вычетов
chore: переработан агент tax-updater
```

## Правила для агентов

- Коммитить только то, что явно попросил пользователь
- Не пушить без подтверждения пользователя
- Стейджить конкретные файлы — не `git add .`
- Всегда добавлять `Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>`
- Не использовать `--no-verify` и `--force`
