# Agent Memory Convention

## Главное правило

После значимой работы — записывай что узнал. Это делает тебя лучше в следующей сессии.

## Когда писать (сразу после):

- Завершил задачу с нетривиальными находками
- Нашёл причину бага
- Принял архитектурное или продуктовое решение
- Потратил на что-то больше 15 минут
- Нашёл подход, который работает (или не работает)

## Где хранить

```
agents/<имя-агента>/memory/YYYY-MM-DD.md
```

Создай директорию если не существует: `mkdir -p agents/<имя>/memory/`

**Всегда в `agents/` (в VCS), никогда в `~/.claude/`.**

## Формат записи

Добавляй секцию к файлу дня (не перезаписывай весь файл):

```markdown
## [HH:MM] Название действия
- Что: краткое описание
- Результат: успех / провал / частично
- Вывод: что запомнить на следующий раз
```

Если файл не существует — создай с заголовком:
```markdown
# YYYY-MM-DD — [Имя агента]
```

## Что записывать

- Что сработало и почему
- Что не сработало и почему — чтобы не повторять
- Специфика проекта: копейки, YTD, две налоговые базы, официальные источники
- Конкретные паттерны: "если видишь X — делай Y"

## Что НЕ записывать

- Содержимое коммитов (уже есть в git)
- Текущий статус задачи (это не постоянная память)
- Предположения без доказательств

## Чтение памяти при старте (обязательно)

Первое действие в новой сессии — прочитать последние 2-3 файла из `agents/<имя>/memory/`.
Если директория не существует — пропусти.

## Планы

Планы реализации хранятся в `project-docs/plans/`.
Перед началом задачи проверь — нет ли готового плана.
