# Code Style - new_tax

Правила для всех агентов, пишущих код в этом проекте.

## Backend (Go)

### Деньги - только в копейках

```go
// ✅ правильно
var salary uint64 = 10_000_000 // 100 000 ₽

// ❌ никогда
var salary float64 = 100000.0
```

Все суммы - `uint64` в копейках. Конвертация: рубли × 100.
Ставки - `uint64` делённые на 1000: 22% → 220, 5.1% → 51.

### Округление налога

Только через `RoundTaxAmount()` из `internal/calculate/`. Никакого ручного округления.
Реализует правило 50 копеек по ст. 52 п. 6 НК РФ.

### YTD - накопительный итог

Налог считается от начала года, месячная сумма = дельта от предыдущего месяца.
Никогда не считай налог за каждый месяц отдельно - это даёт ошибку при переходе через пороги шкалы.

### Где живёт бизнес-логика

**Только в `internal/calculate/`.** Web-слой (`web/`) не считает налог - только отображает.

### Стиль

- Комментарии на русском (как в существующем коде)
- Короткие функции с одной ответственностью
- Не рефакторь код, который не трогаешь
- Не добавляй docstring к существующим функциям без запроса

### Тесты

- Каждая новая ветка расчёта → unit-тест
- Граничные значения обязательны: 0, ровно на пороге, переход через порог
- `testify/assert` и `testify/require`
- После изменений: `go build ./...` и `go test ./...`

### Proto

Изменение `docs/grpc/tax.proto` → обязательно `make codegen`.
Не редактируй файлы в `gen/grpc/` вручную.

## Frontend (Go templates + CSS + JS)

- Шаблоны: `web/templates/*.tmpl` - Go `html/template`
- Стили: vanilla CSS, каждый компонент в своём файле (`web/static/*.css`)
- JS: vanilla JavaScript, `web/static/script.js` - никаких фреймворков
- Форматирование копеек в рубли - через template functions (`web/template_funcs.go`)
- После изменений: `cd web && go build ./...`

## Общее

- Не добавляй абстракции для одноразовых операций
- Не добавляй обработку ошибок для сценариев, которые не могут случиться
- Минимально достаточное решение - не проектируй под гипотетическое будущее
